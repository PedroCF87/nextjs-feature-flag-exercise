## 5) SQL.js Constraints

> Source: T3 Step 5

### Constraint table

| Constraint | Rule | Risk if violated |
|---|---|---|
| Boolean storage | `enabled` stored as INTEGER 0/1 | Filter comparisons fail silently |
| Statement lifecycle | `stmt.free()` must be called in `try/finally` | Memory leak per query |
| Case-insensitive LIKE | Use `LOWER(col) LIKE LOWER(?)` | Case-sensitive results break `name` filter |
| Parameterized queries | Use `db.prepare()` + `stmt.bind()` for any user value | SQL injection risk |
| Dynamic WHERE clause | Build with array push + `AND` join | Never use string interpolation for filter values |
