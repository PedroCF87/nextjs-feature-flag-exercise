
# Delfos AI Orchestrator
## Product Requirements Document (PRD) - v1.0.0
### Agnostic Orchestration Layer for Claude Code & VS Code AI Agents

---

## 1. Executive Summary
The **Delfos AI Orchestrator** is a project-agnostic orchestration platform designed to bridge the gap between AI code generation and human-in-the-loop execution. It provides a centralized dashboard to visualize, manage, and trigger tasks planned by AI agents (Claude Code / Copilot), ensuring transparency and control during complex development cycles.

---

## 2. Goals & Objectives
* **Visibility:** Provide a real-time UI that reflects the AI's internal reasoning and planned sequence of actions.
* **Control:** Enable manual triggering of planned commands to prevent unintended side effects in automated loops.
* **Agnosticism:** Maintain a strict separation between project-specific logic (stored in `CLAUDE.md`) and the orchestration infrastructure.
* **Feedback Loop:** Use webhooks and sockets to stream terminal output directly to the browser.

---

## 3. System Architecture
The system consists of three main components:
1.  **AI Layer (Claude Code):** Responsible for analyzing the project and generating a structured execution plan (JSON).
2.  **Orchestration Backend (Express + Socket.IO):** Acts as the bridge. It receives the plan, manages task states, and executes terminal commands via child processes.
3.  **Control Dashboard (React/Next.js):** The web interface that visualizes the task queue and provides execution buttons.

## 4. Functional Requirements

### 4.1. Structured Planning (AI Agent)
The AI Agent must be instructed to output a `plan.json` format when the `plan` command is invoked.

```json
{
  "planId": "uuid-v4",
  "project": "next-feature-flag-exercise",
  "tasks": [
    {
      "id": 1,
      "description": "Initialize feature flag configuration",
      "command": "npm run init-flags",
      "status": "pending",
      "dependencies": []
    },
    {
      "id": 2,
      "description": "Create middleware for flag detection",
      "command": "touch middleware.ts && code middleware.ts",
      "status": "pending",
      "dependencies": [1]
    }
  ]
}
```

### 4.2. Real-time Feedback (Hooks)
Utilize Claude Code `PostToolUse` hooks to notify the Backend of activity.
* **Endpoint:** `POST /api/webhook/activity`
* **Payload:** Tool name, exit code, output snippet, and duration.

### 4.3. Command Execution Bridge
The Backend implements a secure command execution module using `child_process.spawn` for real-time log streaming.

```javascript
// Backend Logic Sample
socket.on('execute-task', (taskId) => {
    const process = spawn(tasks[taskId].command, { shell: true });
    process.stdout.on('data', (data) => {
        socket.emit('terminal-log', { taskId, data: data.toString() });
    });
});
```

## 6. Security Considerations
> **Warning:** The orchestrator executes shell commands. It must only be accessible via `localhost`. All inputs from the UI must be strictly mapped to the AI-generated `plan.json` to prevent arbitrary command injection.
