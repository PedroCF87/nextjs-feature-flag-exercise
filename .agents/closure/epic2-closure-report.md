## 4 — Decisions Record

| Decision | Rationale | Story |
|---|---|---|
| **`exercise-2` branch from upstream original state** (`f73979ed~1`) | Clean re-implementation without carrying E1 code; enables valid before-after comparison | E2-S2 |
| **Radix SelectItem sentinel value `"all"`** | Empty string `""` crashes Radix `SelectItem` internally (undocumented constraint); `"all"` is a non-empty sentinel filtered in `onChange` | E2-S4 |
| **Direct synchronous ref assignment for `latestFiltersRef`** | Avoids stale-read in debounced callback; ref is updated synchronously during render, not asynchronously in `useEffect` | E2-S4 |
| **PIV Loop local execution with Claude Code Extension** | Claude Code Extension in VS Code provides terminal access, file editing, and the full command system; cloud-only execution was unreliable in E1 | E2-S1 |
| **AND logic for multi-filter composition** | Same rationale as E1: TASK.md example unambiguously requires AND semantics | E2-S3 |
| **Owner exact-match (not LIKE)** | Owner is an identifier (exact match), while name is a search term (partial match via LIKE). Flagged by Claude review; documented as intentional design choice | E2-S4 |
| **Confidence scale 1–5 (not 1–10)** | Workshop uses 1–10 but E1 used 1–5. Keeping 1–5 for apples-to-apples comparison across exercises | E2-S5 |
